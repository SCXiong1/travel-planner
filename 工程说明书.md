## 架构模式

### 三层结构：Route → Service → DB

```
Route（路由层）
  ├── 解析 X-User header
  ├── Pydantic 模型做参数校验
  ├── 调用 Service 处理业务逻辑
  └── 返回 JSON 响应

Service（业务层）
  ├── 包含业务规则和校验
  ├── 直接操作 SQLite
  └── 不处理 HTTP 细节

DB（数据层）
  ├── SQLite 3，WAL 模式
  └── 连接管理集中在 db/database.py
```

### 用户识别

- 前端选身份后存 `localStorage`
- 所有 API 请求带 `X-User: sd` 或 `X-User: sg`
- 后端中间件解析 header，注入 `request.state.user`
- WebSocket 连接时通过查询参数传 `user`

### WebSocket 实时同步

1. 前端进入 Trip 时连接 WebSocket：`ws://host/ws?trip_id=1&user=sd`
2. 服务端维护 `{trip_id: [conn1, conn2]}` 映射
3. 增删改操作完成后，服务端调用 `broadcast(trip_id, event, payload)`
4. 前端收到事件后刷新对应数据
5. 离开 Trip 时断开，服务端清理连接

### 软删除

- 所有表有 `deleted_at` 字段（DATETIME，默认 NULL）
- 删除操作 SET `deleted_at = NOW()`，不做物理删除
- 列表查询默认过滤 `WHERE deleted_at IS NULL`
- 回收站查询 `WHERE deleted_at IS NOT NULL`
- 恢复 SET `deleted_at = NULL`

### 结算算法

```
对每个 Activity 有开销的：
  如果 split = equal：
    sd承担 += amount / 2
    sg承担 += amount / 2
  如果 split = assign：
    payer承担 += amount

结算 = payer支付总额 - 该人承担总额
正数 = 对方欠你，负数 = 你欠对方
```

## 开发环境

### 环境激活（每次新 bash 会话必须执行）

```bash
# 激活 Node.js（fnm 管理）
eval "$(/d/document/fnm/fnm.exe env)"

# Python 使用 myaccounting conda 环境的 Python 直接路径
# conda activate 在 bash 下不可用，直接用完整路径
export PYTHON=/d/document/Anaconda/envs/myaccounting/python.exe
```

所有 Python 命令用 `$PYTHON` 替代 `python`，pip 用 `$PYTHON -m pip`。`uvicorn` 和 `pytest` 在 myaccounting 环境的 Scripts 下，直接用即可。

### 两种开发模式

#### 模式 A：前后端分离（推荐开发时用）

后端和前端各跑各的，前端 Vite dev server 自带 HMR 热更新，请求通过 proxy 转发到后端。

```bash
# 终端1：启动后端
cd server && $PYTHON main.py
# 后端监听 0.0.0.0:3001

# 终端2：首次安装依赖（仅一次）
cd client && npm install

# 终端2：启动前端（带 --host 可从手机访问）
cd client && npm run dev -- --host
# Vite 监听 0.0.0.0:5173，/api 和 /ws 自动 proxy 到 localhost:3001
```

前端 URL：`http://localhost:5173`，手机测试需用 `http://<服务器IP>:5173`

#### 模式 B：后端托管前端（简单测试用）

构建前端静态文件，后端直接 serve。只需要一个端口。重启后端后生效。

```bash
# 1. 构建前端
cd client && npm run build

# 2. 启动后端（会自动 serve client/dist）
cd server && $PYTHON main.py
# 访问 http://<服务器IP>:3001 即可
```

注意：每次改前端代码后需要重新 `npm run build`，并重启后端。

### 局域网手机测试

1. 确保手机和开发机在同一局域网
2. Windows 防火墙需要放行对应端口（5173 或 3001）
3. 获取开发机 IP：`ipconfig` 查看无线局域网适配器 IPv4
4. 手机浏览器访问 `http://<IP>:<端口>`
5. 关于 PWA 安装：`localhost` 下 SW 不注册，局域网 IP 下可正常安装

### 首次环境搭建

```bash
# 后端依赖
cd server && $PYTHON -m pip install -r requirements.txt

# 前端依赖
cd client && npm install

# 构建前端
cd client && npm run build
```

### 测试

```bash
cd server && $PYTHON -m pytest tests/ -v
```

### Docker 构建和运行

```bash
docker build -t travel-planner .
docker run -d -p 3000:3000 -v ./data:/app/data travel-planner
docker compose up -d
```

## 约定

- SQLite 查询用参数化，防止 SQL 注入
- 所有 API 输入用 Pydantic 校验
- 前端 composables 存放跨组件共享状态
- 软删除一律查 `WHERE deleted_at IS NULL`
- 不做物理删除，除非回收站里二次确认删除
- WebSocket 消息格式：`{"type": "activity_created", "payload": {...}}`
- Docker 单容器，FastAPI serve 前端静态文件 + API
