# 08 - 结算页

## Type

AFK

## Blocked by

- 06-活动扩展

## What to build

根据所有活动的开销数据，计算并展示本次旅行的结算结果。

**后端**：聚合查询 Trip 下所有 Activity 的开销数据。按分摊规则计算每人支付总额、每人应承担额、差价。算法：
- `split=equal`：金额 ÷ 2，各承担一半
- `split=assign`：金额全部由支付人承担
- 差价 = 支付总额 − 应承担额（正数=对方欠你，负数=你欠对方）

**前端**：
- 从 Trip 详情页顶部 [结算] 按钮进入 `/trips/:id/settlement`
- 展示：sd 支付总额、sg 支付总额、总开销、每人应承担额
- 差价结论：一行大字 "sg → sd ¥200" 或 "sd → sg ¥XXX"
- 无开销数据时显示"暂无开销记录"

## Acceptance criteria

- [ ] 从行程规划页顶部 [结] 按钮进入结算页
- [ ] 正确显示 sd 和 sg 各自支付总额
- [ ] 正确显示总开销
- [ ] 正确显示每人应承担额
- [ ] 平分的开销各承担一半
- [ ] 归集的开销归到支付人
- [ ] 差价结论直观清晰
- [ ] 无开销时显示无数据提示
- [ ] 返回行程规划页正常
