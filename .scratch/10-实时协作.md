# 10 - 实时协作（WebSocket）

## Type

AFK

## Blocked by

- 05-活动基础管理
- 07-打包清单

## What to build

两人同时打开同一个 Trip 时，一方的修改实时同步给另一方。

**后端**：
- WebSocket 端点 `/ws?trip_id=1&user=sd`
- 连接管理器：`{trip_id: [conn1, conn2]}` 映射
- 广播：增删改 Activity/PackingItem 后，广播事件给同 trip 的其他连接
- 断开自动清理

**前端**：
- 进入 Trip 详情页时连接 WebSocket
- 离开 Trip 时断开
- 监听事件类型：`activity_created`, `activity_updated`, `activity_deleted`, `activity_reordered`, `packing_created`, `packing_updated`, `packing_deleted`, `packing_reordered`
- 收到事件后刷新对应数据（重新请求 API 或直接更新本地状态）

**冲突策略**：最后写入胜出，不做合并检测。

## Acceptance criteria

- [ ] sd 在 Trip A 添加一个活动，sg 的页面即时显示新活动（无需手动刷新）
- [ ] sg 修改活动时间，sd 即时看到
- [ ] sd 删除活动，sg 即时看到活动消失
- [ ] 打包清单同理：增删改勾选即时同步
- [ ] 断开网络后 WebSocket 自动重连
- [ ] 离开 Trip 页面后 WebSocket 断开
- [ ] 两个浏览器窗口可同时测试（或浏览器+隐私窗口）
