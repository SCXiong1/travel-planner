# 02 - 身份选择

## Type

AFK

## Blocked by

- 01-项目骨架

## What to build

最简单的身份识别：不需要密码、不需要注册。

**前端**：一个登录页，两个大按钮 "sd" 和 "sg"。点击后存入 `localStorage`，跳转到旅行列表页。如果已有身份，直接跳转到列表页。

**后端**：一个中间件，读取请求的 `X-User` header，注入到 request state。如果 header 缺失或值不是 sd/sg，返回 401。

**HTTP 客户端**：前端 API 客户端封装，所有请求自动带 `X-User` header。

**WebSocket**：连接时通过查询参数 `?user=sd` 传身份。

## Acceptance criteria

- [ ] 首次访问看到两个按钮 "sd" / "sg"
- [ ] 点击后跳转到旅行列表页
- [ ] 刷新页面不重新选（localStorage 持久化）
- [ ] 不带 X-User header 的 API 请求返回 401
- [ ] 前端 API 客户端自动带 X-User header
- [ ] WebSocket 连接携带 user 参数
