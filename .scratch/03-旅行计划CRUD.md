# 03 - 旅行计划 CRUD

## Type

AFK

## Blocked by

- 02-身份选择

## What to build

旅行计划的创建、查看、编辑、删除。

**API**：trips 表的 RESTful 路由。创建时记录 `created_at`。列表按创建时间倒序。

**前端**：
- 旅行列表页（仪表盘）：显示所有旅行卡片（目的地名、日期范围），点击进入行程规划。空的显示引导提示。
- 创建弹窗：填标题、目的地、开始/结束日期。
- 编辑弹窗：修改上述字段。
- 删除：确认后软删除。

**路由**：`/` 重定向到 `/trips`（列表页），`/trips/:id` 为行程规划页（先占位）。

## Acceptance criteria

- [ ] 创建旅行：填标题+目的地+日期，卡片出现在列表中
- [ ] 列表页显示所有旅行，按创建时间倒序
- [ ] 无旅行时显示引导提示"创建你的第一次旅行"
- [ ] 编辑旅行：修改标题/目的地/日期
- [ ] 删除旅行：确认弹窗，删除后从列表消失（软删除）
- [ ] 点击卡片进入 `/trips/:id`（暂时空白）
