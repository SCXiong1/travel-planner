# 09 - 软删除 + 回收站

## Type

AFK

## Blocked by

- 05-活动基础管理
- 07-打包清单

## What to build

所有删除操作改为软删除（SET deleted_at），并提供一个回收站页面用于查看和恢复已删除项。

**后端**：修改 Activity 和 PackingItem 的 DELETE 路由，SET deleted_at = NOW() 而非物理删除。所有 GET 列表路由过滤 `WHERE deleted_at IS NULL`。新增回收站列表路由和恢复路由。

**前端**：
- 回收站入口（可在 Trip 详情页或设置里）
- 回收站页：显示已删除的活动和打包物品，按删除时间倒序
- 每项显示：名称、类型、删除时间、[恢复] 按钮
- 恢复后从回收站消失，回到原列表
- 回收站为空时显示"回收站为空"

## Acceptance criteria

- [ ] 删除活动后，数据库 deleted_at 有值，不物理删除
- [ ] 活动列表不显示已删除的活动
- [ ] 删除打包物品后同理
- [ ] 回收站页显示所有已删除的活动和物品
- [ ] 点击恢复，deleted_at 变 NULL，物品回到原列表
- [ ] 回收站为空时提示"回收站为空"
