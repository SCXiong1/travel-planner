# 04 - 天管理

## Type

AFK

## Blocked by

- 03-旅行计划CRUD

## What to build

在一个 Trip 下管理多天行程。天有日期和序号，支持拖拽排序。

**API**：days 表的 CRUD + 排序路由。`PUT reorder` 接受 `[{id, day_number}]` 批量更新序号。

**前端**：
- 在行程规划页（`/trips/:id`）左侧渲染天列表
- 每个天显示：day_number（第N天）、日期
- 点击某天高亮选中，右侧显示该天的活动（当前占位）
- 添加天：底部 [+加天] 按钮，默认 day_number 递增
- 拖拽排序天列表（可用 HTML5 drag & drop 或轻量库）
- 删除天：确认弹窗（软删除）

## Acceptance criteria

- [ ] 进入 Trip 详情页，左侧显示天列表（初始为空，有"添加第一天"引导）
- [ ] 添加天：自动分配 day_number，date 跟随之前的天递增
- [ ] 点击某天高亮选中
- [ ] 拖拽更改天顺序后，day_number 自动更新
- [ ] 删除天后从列表消失（软删除）
- [ ] 左侧列表可滚动，太长时不会撑破布局
