# 项目说明书

## 项目概述

travel-planner 是一个家庭旅行规划工具，供 sd 和 sg 两人使用。部署在家庭服务器上（Docker 一键启动），通过浏览器访问，支持手机 PWA 安装（iOS/Android）。

一句话：travel-planner = 行程规划 + 打包清单 + 开销结算 + 实时同步，跑在家庭服务器上。

## 用户场景

1. 出行前：规划每天行程，什么时间去哪、吃什么、住哪里
2. 出行前：制作打包清单，分配谁带什么
3. 旅行中：记录每笔开销，区分谁付的、怎么分摊
4. 旅行中：两个人同时编辑行程，改动实时同步
5. 旅行后：看结算，谁该补给谁多少钱
6. 事后回顾：看每个活动的评价

## 功能模块

### 1. 行程规划

核心模块。结构：Trip → Day → Activity。

- 创建旅行计划（目的地、起止日期）
- 添加天（Day），每天可添加多个活动（Activity）
- 活动分四类：吃 / 住 / 行 / 景点
- 每个活动：名称、位置（可选）、预计开始/结束时间
- 活动按时间排序，支持拖拽调整
- 天列表也支持拖拽排序

### 2. 打包清单

挂在 Trip 下的独立清单。

- 添加物品（名称、分类、负责人 sd/sg）
- 勾选已打包/未打包
- 支持排序

### 3. 开销与结算

开销直接记在 Activity 上，不独立建模块。

- 每个 Activity 可选填：金额、支付人（sd/sg）、分摊方式（平分/归集）
- 结算页：汇总本次旅行总开销、每人支付额、每人应承担额、补差价

### 4. 实时协作

- 两人打开同一个 Trip 时，WebSocket 自动同步
- 一人增删改活动/天/打包项，另一人即时看到
- 冲突策略：最后写入胜出

### 5. 回收站

- 删除为软删除（`deleted_at` 字段）
- 项目和打包物品删除后进入回收站
- 可在回收站恢复

### 6. 简单登录

- 不设密码，不设注册
- 前端点 "sd" 或 "sg" 选择身份
- 后端靠 `X-User` header 识别

## 数据模型

```
Trip
  ├── title
  ├── destination
  ├── start_date, end_date
  ├── Day × N
  │     ├── day_number
  │     ├── date
  │     └── Activity × N
  │           ├── type (eat/stay/transport/sight)
  │           ├── name
  │           ├── location (可选)
  │           ├── start_time, end_time
  │           ├── need_reservation (bool)
  │           ├── reservation_detail (可选)
  │           ├── expense_amount (可选)
  │           ├── expense_payer (sd/sg, 可选)
  │           ├── expense_split (equal/assign, 可选)
  │           ├── review
  │           ├── sort_order
  │           └── deleted_at
  └── PackingItem × N
        ├── name
        ├── category
        ├── assignee (sd/sg)
        ├── checked
        ├── sort_order
        └── deleted_at
```

## 与 TREK 的差异

TREK 是全功能多用户旅行协作平台，travel-planner 是其极简版：

| 特性 | TREK | travel-planner |
|------|------|----------------|
| 用户系统 | 多用户 + 角色 + OIDC + MFA | sd/sg 两按钮 |
| 前端 | React 18 | Vue 3 |
| 后端 | Express (Node.js) | FastAPI (Python) |
| 实时协作 | WebSocket 房间模式 + MFA | WebSocket 简单广播 |
| 额外模块 | 8 个可插拔 addon | 无 |
| AI 集成 | MCP 协议 150+ 工具 | 无 |
| 预算 | 独立模块 + 多币种 | 活动内嵌开销 |
| 日记 | Journey 杂志风格 | 活动评价字段 |
